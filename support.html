AzAd Browser Kit

Visit the browser extension store for your browser (Chrome Web Store, Firefox Add-ons, etc.), search for "AzAd Browser Kit", and click "Add to [Browser]". The extension will install automatically and appear in your extension menu.

AzAd Browser Kit officially supports Chrome, Edge, Firefox, and Opera. Safari support is in development. Check the extension page for your browser to see the latest compatibility information.

Try: 1) Refresh the page, 2) Disable and re-enable the extension, 3) Clear extension data and cookies, 4) Check if the site allows extensions, 5) Restart your browser. If issues persist, report on GitHub Issues with your browser version and error details.

Image Optimizer

Yes. The image-optimizer requires Node.js 14+ and npm. Download Node.js from nodejs.org, then run npm install in the image-optimizer folder to set up dependencies.

Compression depends on the original format and quality. Typically: PNG files compress 30-70%, JPEG files 20-50%, and WebP conversion provides 40-80% additional savings. See actual results in the image-optimizer output logs.

Yes! The image-optimizer is production-ready. It's used in live environments and watches for changes automatically. Test it first in a development environment, then deploy to production with confidence.

Check: 1) Asset folder path is correct in .image-optimizer.json, 2) Image format is supported (PNG, JPG, WebP, SVG, etc.), 3) File has write permissions, 4) Image isn't already optimized, 5) Watch mode is running. Run with --verbose flag for detailed logs.

The image-optimizer can process images up to system memory limits (typically 2-8GB+). No artificial file size limit is enforced. Larger files take longer to process. RAM available on your system is the practical limit.

Form Handlers & CAPTCHA

Visit our form-handlers documentation for a step-by-step guide. You'll need to create a Google Apps Script project, deploy it as a web app, and copy the webhook URL into your form configuration.

Check browser console (F12 > Console tab) for JavaScript errors. Verify Google Apps Script endpoint URL is correct. Test form submission by checking Network tab. Enable verbose logging in form-handler.js. Check FORM-SYNC-CHECKLIST.md for verification steps.

Verify: 1) captcha-class.js is loaded, 2) DOM elements have correct IDs (#captchaCheck, #captchaAnswer, etc.), 3) JavaScript isn't throwing errors (check console), 4) CSS isn't hidden (.captcha-box display property), 5) Form ID matches configuration.

The CAPTCHA supports Material Design styling through CSS classes and inline styles. You can customize colors, sizing, fonts, and animations. See the CAPTCHA-UI documentation for detailed styling options and examples.

Form submissions are sent over HTTPS (encrypted in transit). Data is stored in your Google Drive (if using Google Apps Script). No data is stored on our servers. You control where and how your data is stored.

Yes. Form submissions are stored in your Google Sheet (if you set that up). You can delete responses directly from your Google Sheet. We don't store submission data on our servers.
